# Kith-2.0

A small compiled language that transpiles to C. Clean syntax, fast output, no runtime dependencies.

func main()
{
    str name = "world"
    print($"Hello, {name}!")
}

Requirements: A C++ compiler (g++ / clang++) and gcc on your PATH.

# clone and build
git clone https://github.com/PenguineDavid/Kith-2.0
cd Kith-2.0/src
g++ -std=c++17 main.cpp lexer.cpp parser.cpp codegen.cpp -o kith

Add kith to your PATH or run it directly from the project folder.

kith source.kith              # compiles to ./output
kith source.kith myprogram    # compiles to ./myprogram

# Flags
kith --keep-c source.kith     # keeps the intermediate output.c for debugging
kith --bounds source.kith     # enables runtime bounds checking on array accesses
kith --help

Exceptions are implemented with setjmp/longjmp. An uncaught throw prints to stderr and exits with code 1.

Kith is a transpiler. It compiles your source to C, then calls gcc to produce the final binary. The intermediate output.c file is deleted automatically unless you pass --keep-c.

source.kith  ->  [Lexer]  ->  [Parser]  ->  [AST]  ->  [CodeGen]  ->  output.c  ->  gcc  ->  binary

This means you get fast native binaries with zero runtime overhead, and you can drop in any C library via include "header.h".

        if (i % 15 == 0) {
            print("FizzBuzz")
        } else if (i % 3 == 0) {
            print("Fizz")
        } else if (i % 5 == 0) {
            print("Buzz")
        } else {
            print(i)
        }

Fibonacci

func fib(int n)
{
    if (n <= 1) { return n }
    return fib(n - 1) + fib(n - 2)
}

func main()
{
    for (int i = 0; i < 10; i = i + 1) {
        print(fib(i))
    }
}

Structs and Methods

struct Point
{
    int x
    int y
}

func distance(Point a, Point b)
{
    int dx = a.x - b.x
    int dy = a.y - b.y
    return sqrt(dx * dx + dy * dy)
}

func main()
{
    Point origin = Point(0, 0)
    Point target = Point(3, 4)
    print(distance(origin, target))    // 5
}

Exception Handling

func divide(int a, int b)
{
    if (b == 0) {
        throw "division by zero"
    }
    return a / b
}

func main()
{
    try {
        print(divide(10, 2))    // 5
        print(divide(10, 0))    // throws
    } catch (e) {
        print(e)                // division by zero
    }
}
Flag Description
--keep-c Keep output.c after compilation (useful for debugging generated C)
--bounds Enable runtime bounds checking for array index operations
--help Print usage information
  • All functions emit int as their C return type regardless of what they actually return
  • No static type checking — type errors are caught by gcc, not the Kith compiler
  • Stack-only arrays — no dynamic resizing
  • print(arr[i]) on a float array requires assigning to a local first: float v = arr[i] then print(v)
  • Global array elements must be compile-time constants
  • Proper tracking and freeing of interpolated strings, including globals
  • Python-style for i in range(10) and for item in collection syntax
  • Heap-allocated arrays with dynamic resizing
  • Optional return types: func int add(int a, int b) / func void print(str s)
  • Classes and inheritance mapped to C structs and functions

# lfsr_random

Single-header C++17 PRNG library. Drop one file in, include it, done. No dependencies beyond the standard library.

Copy lfsr_random.hpp into your project. Recommended layout:

your_project/
    include/
        lfsr_random.hpp
    src/
        main.cpp

Include it with:

#include "lfsr_random.hpp"

If you use the recommended layout, pass -I include to the compiler:

g++ -std=c++17 -O2 -I include src/main.cpp -o myapp

Requirements: C++17 or later. No third-party dependencies.

Two engines run under the hood. The API routes to the right one based on the type you ask for.

Engine Class State Period Used for
Fibonacci LFSR lfsr_engine_32 32-bit 2^32 - 1 Int Uint Float Char Bool
xoshiro256** lfsr_engine_128 256-bit 2^256 - 1 Int64 Uint64 Double, all distributions

Both engines are held in thread_local storage. All calls are thread-safe with no mutex overhead. Each thread gets its own independently auto-seeded engine on first use.

#include "lfsr_random.hpp"

int   n = lfsr::Int.uniform(0, 100);       // [0, 100] inclusive
float f = lfsr::Float.uniform(0.f, 1.f);   // [0, 1)
bool  b = lfsr::Bool.uniform();            // 50/50 coin flip
char  c = lfsr::Char.uniform('a', 'z');   // random lowercase letter

std::vector<int> v = {1, 2, 3, 4, 5};
lfsr::shuffle(v.begin(), v.end());

auto it = lfsr::sample(v.begin(), v.end());  // iterator to a random element

lfsr::Int

Backed by the 32-bit LFSR engine.

int lfsr::Int.uniform(int min, int max)
int lfsr::Int.sysTimeSeed()

Returns a uniformly distributed int in [min, max] (both endpoints inclusive). If min >= max, returns min. sysTimeSeed() returns the current RDTSC counter cast to a positive int.

int d6    = lfsr::Int.uniform(1, 6);
int index = lfsr::Int.uniform(0, vec.size() - 1);

lfsr::Uint

Backed by the 32-bit LFSR engine. Same semantics as Int but for unsigned int.

unsigned int lfsr::Uint.uniform(unsigned int min, unsigned int max)
unsigned int lfsr::Uint.sysTimeSeed()

lfsr::Int64

Backed by the xoshiro256** engine. Handles the full signed 64-bit range including ranges that span the sign boundary.

int64_t lfsr::Int64.uniform(int64_t min, int64_t max)

int64_t big = lfsr::Int64.uniform(INT64_MIN, INT64_MAX);
int64_t ts  = lfsr::Int64.uniform(0, 1'000'000'000LL);

lfsr::Uint64

uint64_t lfsr::Uint64.uniform(uint64_t min, uint64_t max)
uint64_t lfsr::Uint64.sysTimeSeed()

uint64_t id  = lfsr::Uint64.uniform(0ULL, UINT64_MAX);
uint64_t off = lfsr::Uint64.uniform(0ULL, file_size - 1);

lfsr::Float

Backed by the 32-bit LFSR engine. 24-bit mantissa precision. Upper bound is exclusive.

float lfsr::Float.uniform(float min, float max)

float t     = lfsr::Float.uniform(0.f, 1.f);
float angle = lfsr::Float.uniform(0.f, 6.2831853f);

lfsr::Double

Backed by the xoshiro256** engine. 53-bit mantissa precision.

double lfsr::Double.uniform(double min, double max)
double lfsr::Double.normal(double mean = 0.0, double stddev = 1.0)
double lfsr::Double.exponential(double lambda = 1.0)

normal() uses a Box-Muller transform; the spare sample is cached per-thread so every two engine calls produce two output samples. exponential() has mean 1 / lambda; lambda must be greater than zero.

double u    = lfsr::Double.uniform(0.0, 1.0);
double roll = lfsr::Double.normal(100.0, 15.0);  // IQ-style distribution
double wait = lfsr::Double.exponential(2.0);     // mean = 0.5

lfsr::Bool

bool lfsr::Bool.uniform()                    // unbiased coin flip
bool lfsr::Bool.bernoulli(double p)          // true with probability p

if (lfsr::Bool.bernoulli(0.05)) trigger_rare_event();

lfsr::Char

char lfsr::Char.uniform(char min, char max)

char lower = lfsr::Char.uniform('a', 'z');
char digit = lfsr::Char.uniform('0', '9');

lfsr::shuffle

Fisher-Yates in-place shuffle over [first, last). Works on any RandomAccessIterator. Empty and single-element ranges are no-ops.

template <typename RandomIt>
void lfsr::shuffle(RandomIt first, RandomIt last)

std::vector<int> deck(52);
std::iota(deck.begin(), deck.end(), 0);
lfsr::shuffle(deck.begin(), deck.end());

int arr[] = {1, 2, 3, 4, 5};
lfsr::shuffle(std::begin(arr), std::end(arr));

lfsr::sample

Returns an iterator to a uniformly random element in [first, last). Returns last if the range is empty — always check before dereferencing.

template <typename RandomIt>
RandomIt lfsr::sample(RandomIt first, RandomIt last)

auto& winner = *lfsr::sample(players.begin(), players.end());

auto it = lfsr::sample(v.begin(), v.end());
if (it != v.end()) use(*it);

lfsr::weighted_pick

Picks a value from a table of {value, weight} pairs. Weights are relative and do not need to sum to 1. All weights must be non-negative with at least one greater than zero. Throws std::invalid_argument if the table is empty.

template <typename T>
T lfsr::weighted_pick(const std::vector<std::pair<T, double>>& table)

using P = std::pair<std::string, double>;
auto loot = lfsr::weighted_pick<std::string>({
    P{"common sword",  70.0},
    P{"rare shield",   25.0},
    P{"epic axe",       4.5},
    P{"legendary bow",  0.5},
});

lfsr::weighted_index

Same as weighted_pick but returns the index into the weight vector rather than a value.

std::size_t lfsr::weighted_index(const std::vector<double>& weights)

std::vector<double> spawn_weights = {10.0, 5.0, 1.0};
std::size_t enemy_type = lfsr::weighted_index(spawn_weights);

Engines are auto-seeded from RDTSC (or std::chrono::steady_clock on non-x86) mixed with a stack address on first use. You only need to seed manually for reproducible output.

// Seed the 32-bit engine (affects Int, Uint, Float, Char, Bool)
lfsr::seed_32(42u);

// Seed the 128-bit engine (affects Int64, Uint64, Double, distributions)
lfsr::seed_128(0xDEADBEEFu, 0xCAFEBABEu);

Seeding is per-thread. Calling seed_32 on thread A does not affect thread B's engine.

int main() {
    lfsr::seed_32(12345u);
    lfsr::seed_128(12345u, 67890u);
    // all subsequent calls are deterministic on this thread
}
lfsr_engine_32&  e32  = lfsr::engine32();
lfsr_engine_128& e128 = lfsr::engine128();

uint32_t raw32 = e32();
uint64_t raw64 = e128();

// Both satisfy UniformRandomBitGenerator - work with std:: algorithms
std::shuffle(v.begin(), v.end(), lfsr::engine32());
std::generate(v.begin(), v.end(), lfsr::engine32());

// Local instances without touching thread_local state
lfsr_engine_32  local32(0xABCDu);
lfsr_engine_128 local128(0x1234u, 0x5678u);

for (int i = 0; i < 100; ++i)
    process(local32());

Every thread has its own engine instances via thread_local. There are no shared mutable globals and no locks. Calls from different threads never interfere.

std::vector<std::thread> workers;
for (int i = 0; i < 8; ++i)
    workers.emplace_back([]() {
        int v = lfsr::Int.uniform(0, 100);  // independent from all other threads
    });

Note that seed_32 and seed_128 only affect the calling thread's engine.

The test suite is in tests/test_lfsr_random.cpp. From the project root:

g++ -std=c++17 -O2 -Wall -Wextra -pthread -I include tests/test_lfsr_random.cpp -o test_lfsr
./test_lfsr

Expected output ends with 52 / 52 tests passed.

All integer samplers use rejection sampling to eliminate modulo bias. The 32-bit sampler uses the Lemire fast-range method with a rejection loop; the 64-bit sampler mirrors the same approach. Floating-point outputs use the top 24 bits (float) or top 53 bits (double) of the raw engine output, mapping directly to the mantissa width of each type.
