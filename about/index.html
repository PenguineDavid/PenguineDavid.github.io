About

I'm David, a full-stack developer and systems programmer based in NSW australia. I started programming through hardware - circuits, logic gates, and eventually building my own CPU in Minecraft redstone - and that low-level foundation has shaped everything I've written since.

What I Like To Build

Systems & Native Tools

Most of my C and C++ work is low level - terminal emulators, system utilities, and low-level tooling for Windows and Linux. I prefer fully static binaries with as few as possible dependencies. On the side I've experimented with compiler and language design, building a small language of my own that transpiles to C.

Redstone & Computer Architecture

For Anyone who doesn't know, redstone is a Minecraft material that can transmit power, and a redstone torch can be used to create a NOT gate. With enough redstone and some clever engineering you can build adders, multipliers, and even entire CPUs inside Minecraft. Computational redstone is where I started. I learned about binary, logic gates, and computer architecture.

Minecraft Server Development

I develop SpigotMC plugins in Java, tho I'm not particularly good at Java itself.

Background

My entry point into software and hardware was computational redstone. Building logic gates, adders, and eventually a full CPU in Minecraft gave me a good intuition for binary, machine code, and computer architecture before I'd written a single line of code. That background means I'm comfortable reasoning about what's happening at the hardware level even when I'm writing high-level code.

From there I moved into JS, then picked up HTML, CSS, and eventualy I came across a Python coding game, and I finnaly experienced programming in a more traditional sense. I dabbled in Python for a while, but I was more interested in learning about how software interacts with hardware, so I moved on to C and C++. Since then I've picked up other languages and technologies.
